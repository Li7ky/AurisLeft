# ⚙️ 战区规范：Agent-Gamma (底层扫描链路开发)

## 身份定位
- **代号**: Agent-Gamma
- **职能**: 系统集成专家 / Native 架构师
- **核心目标**: 打破浏览器沙箱，利用 Tauri 底层能力实现本地音频文件的高效扫描与元数据提取。

## 工作范围 (修改地图)
1. `src-tauri/src/commands/local_music.rs`: 编写 Rust 指令，实现文件树遍历与元数据解析。
2. `src-tauri/Cargo.toml`: 引入音频解析相关 crate (如 `id3` 或 `lofty`)。
3. `src/hooks/useLocalMusic.ts`: 封装 Tauri IPC 通信，向前端暴露数据。
4. `src/pages/LocalMusic/index.tsx`: 本地音乐页面的数据绑定与展示。

## 任务执行清单
- [ ] **底层链路搭建 (Rust)**:
  - 实现 `scan_local_music_dir` Command，支持用户选择目录。
  - 使用异步流或线程池遍历目录结构，匹配 `.mp3`, `.flac`, `.wav` 等扩展名。
  - 提取 ID3 标签（歌曲名、歌手、专辑名、时长、内嵌封面）。
- [ ] **IPC 桥梁构建 (TS)**:
  - 通过 `@tauri-apps/api/invoke` 调用 Rust Command。
  - 建立前端状态缓存机制，避免每次进入页面重复扫描。
- [ ] **异常与性能处理**:
  - 大文件目录扫描时的分页/流式返回机制，防止阻塞 UI 主线程。
  - 处理无权限、文件损坏等异常，包装为统一格式抛出给前端。

## 铁律约束 (参考泰坦机魂 v10.0)
- **第七铁律 (命令必终止)**: 扫描目录必须支持取消机制或超时熔断，禁止出现无响应的永久等待。
- **并发状态管理 (C-1)**: 如果扫描耗时过长，需向前端 emit 进度事件，前端必须正确消费更新。
- **降级处理**: 当 ID3 标签缺失时，使用文件名为备用展示名，禁止触发空指针。

## 完成标准
在本地音乐页面，点击“扫描目录”，能弹出系统文件选择框。选中目录后，列表平滑加载出音频文件，并能看到解析出的歌名与歌手。
