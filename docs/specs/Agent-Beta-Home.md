# ⚙️ 战区规范：Agent-Beta (首页内容矩阵开发)

## 身份定位
- **代号**: Agent-Beta
- **职能**: UI/UX 铸造师
- **核心目标**: 建立一套工业级的首页内容展示系统，将枯燥的页面转化为充满活力的发现中心。

## 工作范围 (修改地图)
1. `src/pages/Home/index.tsx`: 主页面结构重组。
2. `src/components/common/MediaCard.tsx` (需新建): 通用的媒体项展示组件。
3. `src/components/home/HomeSection.tsx` (需新建): 逻辑解耦的分区组件。
4. `src/pages/Home/index.css`: 实现响应式网格布局。

## 任务执行清单
- [ ] **视觉系统构建**: 
  - 实现“发现”、“榜单”、“歌单”等核心 Tab 切换。
  - 采用 `Grid Layout` 实现响应式媒体卡片矩阵，支持不同分辨率。
- [ ] **组件工程化**:
  - 创建 `MediaCard` 组件，支持悬浮播放按钮、渐变阴影及图片懒加载。
  - 增加 Skeletion Screen（骨架屏）以优化加载体验。
- [ ] **数据灌注**: 
  - 定义首页 Mock 数据结构（歌单名、封面图、播放量）。
  - 对接 `playlistStore`，实现点击卡片进入歌单详情。
- [ ] **动效优化**:
  - 引入 `framer-motion` 或 CSS 关键帧实现平滑的入场动画。

## 铁律约束 (参考泰坦机魂 v10.0)
- **UI 一致性**: 必须严格遵循 `global.css` 定义的设计系统（颜色、间距、半径）。
- **性能防线**: 网格列表需考虑虚拟滚动或合理限制首屏渲染数量，防止内存过度消耗。
- **验证闭环**: 每个重构后的分区必须在多种分辨率（900x600 为基准）下完成截图校验。

## 完成标准
首页呈现至少三个分区（如：热门歌单、新碟首发、个性推荐），卡片布局整齐，悬浮效果灵敏。
